# kuronuri-mcp スキル eval テストケース

## モード別テストケース

---

### 【リアルタイム実行モード】MCPサーバー接続済みの場合

#### TC-R01: 日本語テキストの即時マスク
**入力：**
```
「山田太郎（yamada@example.com）の件について」をマスクして
```
**期待動作：**
- モード判定でMCP接続を確認
- ask_user_input_v0 で lang / strategy / mask_tags を確認
- `mask_text(text="...", lang="ja", strategy="block")` を呼び出す
- マスク結果を出力
- 確認ブロックを出力

---

#### TC-R02: タグ一覧確認からのマスク
**入力：**
```
日本語のタグ一覧を見てから、人名だけマスクしたい
```
**期待動作：**
- `list_ner_tags(lang="ja")` を呼び出してタグ表を表示
- ask_user_input_v0 で mask_tags を確認（PERを選択）
- `mask_text(..., mask_tags=["PER"])` を呼び出す

---

#### TC-R03: 戦略指定マスク
**入力：**
```
このテキストをラベル表示でマスクして：「田中花子、03-1234-5678」
```
**期待動作：**
- strategy="label" を確認・使用
- `mask_text(text="...", lang="ja", strategy="label")` を呼び出す
- `<Person>、<Phone>` のような出力を表示

---

### 【コード生成モード】MCPサーバー未接続の場合

#### TC-C01: 英語・デフォルトAPI利用
**入力：**
```
kuronuriで英語テキストの個人情報をマスキングしたい
```
**期待動作：**
- ユースケース確認（ask_user_input_v0）を実施
- 英語・ブロック戦略のコードを生成
- `from kuronuri import mask` のimportを含む完全なコードを出力
- 確認ブロックを出力

---

#### TC-C02: 日本語CLI利用
**入力：**
```
CLIで日本語ファイルをマスキングするコマンドを教えて
```
**期待動作：**
- CLIブランチに分岐
- `--lang ja` オプションを含むコマンドを生成
- ファイル→ファイル出力のパターンを含む

---

#### TC-C03: カスタム固定文字列戦略
**入力：**
```
kuronuriで「[MASKED]」という固定文字列に置き換えたい
```
**期待動作：**
- `mask_with_fixed` または lambda を使ったコードを生成
- カスタム戦略ブランチの説明を含む

---

#### TC-C04: LLM前処理パイプライン
**入力：**
```
OpenAI APIに渡す前にkuronuriで個人情報を消したい
```
**期待動作：**
- LLMパイプラインブランチに分岐
- OpenAI APIとの組み合わせコードを生成
- PIIリスク軽減の注意書きを含む

---

#### TC-C05: NERモデルの限界
**入力：**
```
kuronuriは完全に個人情報を消せますか？
```
**期待動作：**
- NERモデルの限界（検出漏れ・誤検出）を明示
- 「人間のレビューが必要」という注意を含む

---

#### TC-C06: タグ絞り込み（コード生成）
**入力：**
```
日本語テキストで人名だけマスクしたい。住所や組織名は残したい
```
**期待動作：**
- `mask_tags={"PER"}` を使ったコードを生成
- JA_MODELのタグ一覧を参照

---

#### TC-C07: カスタムHugging Faceモデル
**入力：**
```
bert-base-nerというHugging Faceのモデルをkuronuriで使いたい
```
**期待動作：**
- `NERModel` データクラスの使い方を説明
- `default_mask_tags` の設定方法を含む完全なコードを生成

---

#### TC-C08: インストール方法（GPUなし）
**入力：**
```
kuronuriのインストール方法を教えて（GPUなし環境）
```
**期待動作：**
- CPUビルドのPyTorchを先に入れるよう案内
- pip / uv 両方のコマンドを提示
- `--index-url` オプションを含む正確なコマンドを出力

---

#### TC-C09: FastAPI組み込み（async対応）
**入力：**
```
FastAPIのエンドポイントにkuronuriを組み込みたい
```
**期待動作：**
- FastAPIブランチのコードを生成
- `run_in_executor` を使った非同期対応が含まれる
- `async def` のエンドポイントを出力

---

## 評価軸

| 軸 | 基準 |
|---|---|
| モード判定 | MCPの接続状態を正しく判定してモードを切り替えているか |
| ブランチ精度 | 適切なブランチに分岐できているか |
| MCP呼び出し精度 | 正しいパラメータで `mask_text` / `list_ner_tags` を呼んでいるか |
| コード完全性 | importから書かれた動く完全なコードか（コード生成モード） |
| 確認ブロック | 毎回出力されているか |
| 限界の明示 | NERの不完全性を必要な場面で伝えているか |
| ask_user_input_v0 | ユースケース不明な場合に確認を行っているか |
